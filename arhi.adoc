= Архитектура системы управления вентилятором
:toc: auto
:toclevels: 2

image::arhitec.jpg[align="center", title="Схема архитектуры"]

== Что делает эта программа?

Это система, которая **сама управляет вентилятором** в зависимости от температуры в комнате.

- Если **холодно** (менее 23°C) → вентилятор **выключен**
- Если **нормально** (23-35°C) → вентилятор крутится **быстрее или медленнее** (плавная регулировка)
- Если **жарко** (более 35°C) → вентилятор работает **на полную мощность**


== Главные части системы

=== 1. Задачи (помощники) которые работают одновременно

[cols="1,3,2,2"]
|===
| Имя | Что делает | Как часто | Приоритет
| **SensorTask** | Измеряет температуру, фильтрует, рассчитывает скорость вентилятора | Каждые 100 мс | 3 (самый высокий)
| **DisplayTask** | Показывает температуру и скорость на LCD дисплее | Каждые 200 мс | 2
| **LedTask** | Управляет светодиодами (ШИМ + пороговое включение) | Каждые 100 мс | 2
| **UartTask** | Отправляет данные на компьютер для логирования | Каждую 1000 мс | 1 (самый низкий)
|===

=== 2. Общая записная книжка (SharedData)

Все задачи читают и пишут через одну структуру:

[source,cpp]
----
struct SharedData {
    float temperature;   // текущая температура (градусы Цельсия)
    float speed;         // скорость вентилятора (0-100%)
    bool new_data;       // флаг: появились ли новые данные
};
----

=== 3. Компоненты системы

==== Датчик (BME280)

- Измеряет температуру в комнате
- Подключён по шине I2C (пины PB8, PB9)
- I2C адрес: `0x76`
- Методы: `Init()`, `ReadTemperature()`

==== Фильтр низких частот (LowPassFilter)

Сглаживает резкие скачки температуры (экспоненциальный фильтр).

Формула:

`τ = 1 − e^(−dt/RC)`

`y[n] = y[n−1] + (x[n] − y[n−1]) × τ`

где:

- `x[n]` — сырое значение с датчика
- `y[n-1]` — предыдущее отфильтрованное значение
- `y[n]` — новое отфильтрованное значение
- `dt = 0.1 с` — период дискретизации
- `RC = 2.0 с` — постоянная времени

==== Калькулятор скорости вентилятора (SpeedCalculator)

Линейный закон управления:

[source]
----
T <= 23°C  → speed = 0%
T >= 35°C  → speed = 100%
23°C < T < 35°C → speed = (T - 23) / (35 - 23) * 100

Пример: T = 29°C → speed = (29-23)/12*100 = 50%
----

==== Калькулятор скорости для светодиодов (SpeedLedCalculator) (добавлен)

Рассчитывает скважность ШИМ для LED1:

[source]
----
duty = speed / 100 * 999
----

где duty — значение в регистр TIM2_CCR1 (0-999).

==== Форматтер дисплея (DisplayFormatter)

Преобразует числа в строки для вывода на LCD:

[source]
----
25.3 → "T: 25.3 C"
75 → "Speed: 75%"
----

==== Форматтер UART (UARTFormatter)

Преобразует числа в строки для отправки по UART:

[source]
----
25.3 → "Temperature: 25.3 C\r\n"
75 → "Speed: 75 %\r\n"
----

==== Драйвер LCD (LiquidCrystal_I2C)

Низкоуровневый драйвер для LCD дисплея через I2C:
- Адрес: `0x27`
- Размер: 16x2 символов
- Протокол: 4-битный режим HD44780

==== Светодиоды (LEDIndicator)

4 светодиода для визуальной индикации скорости:

[cols="1,2,2,3"]
|===
| LED | Пин | Режим | Условие
| LED1 | PA5 | ШИМ (TIM2_CH1) | Яркость = speed / 100 * 999
| LED2 | PC9 | Дискретный | speed >= 30%
| LED3 | PC8 | Дискретный | speed >= 60%
| LED4 | PC5 | Дискретный | speed >= 90%
|===

==== Логгер (UARTLogger)

Отправка данных на компьютер:
- USART2, пин PA2 (TX)
- Скорость: 9600 бод
- Формат: 8 бит, 1 стоп-бит, без чётности
- Вывод: `Temperature: 25.3 C` и `Speed: 75 %`


== Потоки данных (как всё работает)

=== SensorTask (100 мс, приоритет 3)

[source]
----
1. BME280.ReadTemperature() → raw_temp (23.5°C)
2. LowPassFilter.Update(raw_temp) → filtered_temp (23.4°C)
3. SpeedCalculator.Calculate(filtered_temp) → speed (3%)
4. SharedData.temperature = filtered_temp
5. SharedData.speed = speed
6. SharedData.new_data = true
----

=== DisplayTask (200 мс, приоритет 2)

[source]
----
1. if (SharedData.new_data)
2.   temp = SharedData.temperature
3.   speed = SharedData.speed
4.   DisplayFormatter.FormatTemperature(temp) → "T: 23.4 C"
5.   DisplayFormatter.FormatSpeed(speed) → "Speed: 3%"
6.   LcdDisplay.ShowTemperature() → вывод на экран
7.   LcdDisplay.ShowSpeed() → вывод на экран
----

=== LedTask (100 мс, приоритет 2)

[source]
----
1. speed = SharedData.speed
2. SpeedLedCalculator.Calculate(speed) → duty = 3/100*999 = 29
3. TIM2_CCR1 = duty (LED1 яркость)
4. if (speed >= 30) → LED2 ON, else OFF
5. if (speed >= 60) → LED3 ON, else OFF
6. if (speed >= 90) → LED4 ON, else OFF
----

=== UartTask (1000 мс, приоритет 1)

[source]
----
1. temp = SharedData.temperature
2. speed = SharedData.speed
3. UARTFormatter.FormatTemperature(temp) → "Temperature: 23.4 C\r\n"
4. UARTFormatter.FormatSpeed(speed) → "Speed: 3 %\r\n"
5. UARTLogger.PutString() → отправка через USART2
----


== Почему такие приоритеты и периоды?

[cols="2,2,3"]
|===
| Компонент | Приоритет | Почему
| SensorTask | 3 (самый высокий) | Управление вентилятором не должно задерживаться
| DisplayTask | 2 | Экран важен, но может подождать
| LedTask | 2 | Светодиоды важны, но не критичнее датчика
| UartTask | 1 (самый низкий) | Логирование — некритичная функция, может быть медленным
|===

[cols="2,2,3"]
|===
| Задача | Период | Почему
| SensorTask | 100 мс | Датчик опрашивается 10 раз в секунду — достаточно для плавного управления
| DisplayTask | 200 мс | Глаз не замечает обновления 5 раз в секунду
| LedTask | 100 мс | ШИМ частотой 10 Гц достаточно для плавности яркости
| UartTask | 1000 мс | Отправка раз в секунду — достаточно для записи логов
|===

== Аппаратные подключения

[source]
----
BME280    : SCL=PB8, SDA=PB9 (I2C, addr 0x76)
LCD 16x2  : SCL=PB8, SDA=PB9 (I2C, addr 0x27)
LED1      : PA5 (TIM2_CH1, PWM)
LED2      : PC9 (GPIO output)
LED3      : PC8 (GPIO output)
LED4      : PC5 (GPIO output)
USART2 TX : PA2 (UART, 9600 baud)
----

== Итог

Система работает на 4 параллельных задачах. Данные передаются через SharedData. Датчик опрашивается 10 раз в секунду, экран обновляется 5 раз в секунду, светодиоды — тоже 10 раз в секунду (достаточно для плавного ШИМ), логирование — раз в секунду. Вентилятор управляется по линейному закону: холодно — выключен, жарко — на полную, между — плавная регулировка.
